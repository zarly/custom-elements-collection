# ADR-010 — Deprecate `<ce-stat-group>` in favor of `<ce-grid>` + `<ce-kpi>`

**Status:** Accepted (2026-05-18).
**Supersedes:** none.
**Relates to:** [ADR-002](adr-002-light-dom.md) (Light DOM), [ADR-006](adr-006-component-tier.md) (`stability: "deprecated"` enum), [ADR-009](adr-009-llm-tolerant-components.md) (LLM-tolerant components), [CDR-006](../cdr/cdr-006-components-compose.md), [CDR-008](../cdr/cdr-008-additive-changes-only.md).

## Context

`<ce-stat-group>` shipped as a layout-tier wrapper specialized for KPI tiles: a CSS-grid container with a small props surface (`columns`, `gap`) and a default slot intended for `<ce-kpi>` children. Adoption audit findings (2026-05-17):

- **0 corpus adopters** of `<ce-stat-group>`.
- **28 corpus adopters** of `<ce-grid>` + `<ce-kpi>` covering the same shapes (`<ce-grid columns="3"><ce-kpi …/></ce-grid>`).

The pattern that won was the more general one: `<ce-grid>` accepts arbitrary children (a narrative `<p>` annotation, a `<ce-callout>` warning, a mixed-typology row), while `<ce-stat-group>` was conceptually "KPI-only". In the moments when authors actually wanted to mix component types, `<ce-stat-group>`'s implied narrower contract pushed them to `<ce-grid>` instead — and once there they had no reason to switch back.

This is the exact pattern [CDR-006](../cdr/cdr-006-components-compose.md) codifies: components should accept arbitrary children unless ARIA mandates a specific child role. A "stat group" wrapper that filters or pretends to filter children loses to a generic grid that doesn't.

## Decision

Mark `<ce-stat-group>` deprecated and remove it in the next minor release. Specifically:

1. Set `meta.json` `stability: "deprecated"` (this ADR).
2. Set `deprecatedIn: "0.4.0"` and `replacedBy: "ce-grid + ce-kpi"`.
3. Prefix the meta `description` with `"**Deprecated** — use <ce-grid> with <ce-kpi> children instead. Will be removed in next minor (see ADR-010)."` so it surfaces in `dist/registry.json` and the catalog block of `skill/SKILL.md`.
4. Keep the source, tests, and examples shipping for **≥ 1 quarter** ([CDR-008](../cdr/cdr-008-additive-changes-only.md) soft-keep) so LLM-emitted markup that uses the old tag keeps rendering.
5. Remove the component in the **next minor** release via a follow-up ADR that references this one. The removal commit will also drop the tag from `src/index.ts`, `src/auto.ts`, `src/entries/*`, `src/manifest*.ts`, and `package.json#exports` (all auto-regenerated by `pnpm gen-exports`).

## Migration recipe

```html
<!-- Before (still works during soft-keep) -->
<ce-stat-group columns="3">
  <ce-kpi value="$12 480" label="MRR" trend="+18%"></ce-kpi>
  <ce-kpi value="142ms"   label="P95" trend="-9ms"></ce-kpi>
  <ce-kpi value="98.4%"   label="Uptime"></ce-kpi>
</ce-stat-group>

<!-- After (preferred) -->
<ce-grid columns="3">
  <ce-kpi value="$12 480" label="MRR" trend="+18%"></ce-kpi>
  <ce-kpi value="142ms"   label="P95" trend="-9ms"></ce-kpi>
  <ce-kpi value="98.4%"   label="Uptime"></ce-kpi>
</ce-grid>
```

Codemod: simple s/ce-stat-group/ce-grid/g on attribute-identical use sites. Document this in the migration section of `skill/SKILL.md` when the removal lands.

## Consequences

- ✅ Catalog shrinks by one component; no observed corpus impact (0 adopters).
- ✅ Composability principle ([CDR-006](../cdr/cdr-006-components-compose.md)) is reinforced — generic containers win over specialized ones unless ARIA mandates otherwise.
- ✅ Consistent with the LLM-tolerant doctrine ([ADR-009](adr-009-llm-tolerant-components.md)): we'd rather a generic component accept a slightly-off-pattern child than a specialized one silently drop it.
- ⚠️ LLM markup that still emits `<ce-stat-group>` will work during the soft-keep window. We accept this transient bloat in exchange for honouring the no-silent-breakage contract.
- ⚠️ The catalog now lists a `deprecated` component until removal; downstream registry consumers should respect `stability` in queries (already part of the registry schema per [ADR-007](adr-007-component-registry.md)).

## Validation

- `pnpm validate-meta` keeps passing — `deprecated` is a valid stability value per ADR-006.
- The registry generator (`pnpm gen-registry`) propagates the new `stability` into `dist/registry.json` / `dist/registry/by-tier/layout.json`.
- The `cec-component-validator` rule R10 (overlap detection) already flags `ce-stat-group` as overlapping with `ce-grid + ce-kpi`; this ADR retires the entry from the rule's overlap allowlist on removal.
